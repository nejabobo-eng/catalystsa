# Product Copy Template - High-Conversion Format

## 🎯 PRODUCT CONVICTION FRAMEWORK

Every product description must answer 3 questions:
1. **What is it?** (Quick positioning)
2. **What do I get?** (Benefit bullets, not specs)
3. **Who is it for?** (Clear target customer)

---

## 📝 TEMPLATE STRUCTURE

### Product Name Format:
```
[Brand/Type] [Model] – [Key Benefit Phrase]
```

**Examples:**
- ✅ Dell Inspiron 15 – Fast Reliable Laptop for Work & School
- ✅ Nike Air Max – Comfortable All-Day Sneakers
- ❌ Dell Inspiron 15 Laptop (boring, no benefit)

---

### Description Format:

```
[One-line outcome statement]

• [Benefit 1 with outcome]
• [Benefit 2 with outcome]
• [Benefit 3 with outcome]

Perfect for:
- [Target customer 1]
- [Target customer 2]
- [Target customer 3]
```

## 🧠 WRITING RULES (CRITICAL)

### ✅ DO THIS:
- Start with **outcomes** ("Fast performance" not "Intel i5")
- Use **benefit bullets** ("comfortable for all-day wear" not "cushioned sole")
- Name **specific use cases** ("students, office work" not "anyone")
- Keep it **scannable** (bullets, short paragraphs)
- Use **SA language** (natural, not overly formal)

### ❌ DON'T DO THIS:
- Spec dumps ("15.6-inch, 8GB, 512GB...")
- Marketing fluff ("revolutionary", "game-changing")
- Vague claims ("high quality", "great product")
- Generic targets ("everyone will love this")
- Too long (over 100 words)

## 📊 BENEFIT vs SPEC TRANSLATION

| ❌ SPEC (What it is) | ✅ BENEFIT (What you get) |
|---------------------|---------------------------|
| Intel i5 processor | Fast performance, smooth multitasking |
| 8GB RAM | Run multiple programs without slowing down |
| 512GB SSD | Store all your files with quick access |
| Cushioned sole | Comfortable for all-day wear |
| Breathable fabric | Keeps feet cool and fresh |
| Lightweight design | Easy to carry, no strain |

## 🎯 TARGET CUSTOMER FRAMEWORK

Always name **3-4 specific groups**:

### Good Examples:
- Students (assignments, online classes, research)
- Office workers (emails, documents, video calls)
- Small business owners (accounting, invoicing, client communication)
- Parents (online shopping, family planning, entertainment)
- Remote workers (Zoom meetings, file storage, multitasking)
- Fitness enthusiasts (tracking workouts, music, motivation)
- Casual users (browsing, social media, streaming)

### Bad Examples:
- "Anyone" (too vague)
- "Professionals" (not specific enough)
- "People who need laptops" (circular logic)

3. Update **Name** (add benefit phrase)
4. Update **Description** (use template format)
5. Save → Changes appear instantly on storefront

- [ ] Description starts with outcome statement
- [ ] 3 benefit bullets included
